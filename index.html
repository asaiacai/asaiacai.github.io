Recover Homographies

Homography recovery requires at least 4 different correspondence points since homography has 8 degrees of freedom. However, having an overdetermined system and solving through least squares results in a much more robust estimate of the the homography transform. Image Warp

Now that we have a homography matrix, H, we can begin coloring. To do this, I first applied H on vertices of the source image to find the bounding polygon of the transformed image. From here, I can find out which pixels I need to color in the destination. I calculate the inverse transform to back transform the pixel coords back onto the source texture for sampling through interpolation. We also set the alpha value for the colored pixels to 1. We can use homography to rectify images so that for images of planar surfaces that we take from an angle, we can see what they would look like from a perpendicular viewpoint. For this, we found some squares the vertices serving as the correspondence points homography mapping from these points to the Cartesian product of {0, 1} x {0, 1}

Part B : Feature Matching for Autostitching

Step 1 : Harris Corner Detection

To begin detecting features, we look for corners which are points in an image that sharply change intensity along the corner directions. To do this we use the Harris Corner detector using the sample code provided. The way the Harris Corner detector works is it compares the relative change in intensity in some window in an image against a shifted window. If the change in intensity is sharp along all directions, we know that this location contains a corner. Below is an image plotted with the detected corners

Step 2 : Adaptive Non-Maximal Suppression

We can detect corners but many of the keypoints are densely packed together. This is not so useful for computing homography so we should decrease the density whilst keeping the most important features. We do Adaptive Non-Maximal Suppression, which just means we keep the strongest points (with respect to the corner response function) within the locality of the point. We do this by setting the suppression radius to infinity initially. Initially, at this point we only keep is the global max corner. However, as we decrease the suppression radius, more locally maximal points will be added until we get n = 500 points. Below is an plot of the same image using adaptive non-maximal suppression to filter keypoints out

Step 3 : Feature Descriptor extraction and matching

To finalize, our feature finding, we need to create descriptors for each keypoint. To do this, we just take a (40x40) axis-aligned box centered about the keypoint then downsample it to (8x8). We bias/gain normalize this descriptor by subtracting away the mean and dividing by the standard deviation and save this downsampled image as a feature descriptor vector.

Now that we have a set of keypoints and associated descriptors, we can try matching keypoints between two images using the feature descriptors. To match features, we just use nearest neighbor search in the feature space. However, not all feature pairs are likely to be that useful. So we filter by thresholding matches with respect to the ratio of the first-nearest neighbor match to the second nearest neighbor match, the idea being that a good feature pair should unambiguously match one point in each image. Below is a plot of the matched feature, a pair of two images.

Step 4 : RANSAC homography estimation

Even after filtering and keeping the best features we could find, there's a good chance of bad matches. To exclude these outliers from the homography estimation, we use Random Sample Consensus (RANSAC). Random sample consensus works by randomly picking 4 features pairs to compute a homography. Using this randomly generated homography estimate, we transform the set of keypoints and find which points are inliers, which are just the keypoint pairs that are below some threshold distance apart after transformation. If enough inliers are detected (I required at least 8 points), we compute another homography using all the inliers found, calculate the MSE on this new homography, and keep the homography if it gives a lower MSE on the inlier set. We repeat this process for multiple iterations and return the homography thaat gave us the smallest inlier MSE.
