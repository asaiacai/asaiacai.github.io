Overview

The goal of this project is to create image rectifications and image mosaics through calculation of homographies.

Recover Homographies

Homography recovery requires at least 4 different correspondence points since homography has 8 degrees of freedom. However, having an overdetermined system and solving through least squares results in a much more robust estimate of the the homography transform. I implemented the homography estimation through spectral value decomposition as outlined here. Then I just had to choose my corresondence points which I illustrate below

Image Warp

Now that we have a homography matrix, H, we can begin coloring. To do this, I first applied H on vertices of the source image to find the bounding polygon of the transformed image. From here, I can find out which pixels I need to color in the destination. I calculate the inverse transform to back transform the pixel coords back onto the source texture for sampling through interpolation. We also set the alpha value for the colored pixels to 1. Below is the intermediate result of some of my warps before stitching.

Image Rectification

We can use homography to rectify images so that for images of planar surfaces that we take from an angle, we can see what they would look like from a perpendicular viewpoint. For this, we found some squares the vertices serving as the correspondence points homography mapping from these points to the Cartesian product of {0, 1} x {0, 1}

Image blending

We can also create panoramas by blending projective transformed images together. For me it was sufficient enough to warp one image onto an unwarped destination image. I would have a blank canvas big enough to fit the combined images after which I would add them together. For combining images, alpha blend averaging the two images together worked best for me in reducing linear artifacts. My results are shown below

From this project I learned that getting calculating the homography requires a lot of really spaced out points. For part of the project, I had chosen correspondence points that were too close together giving me a really bad H matrix which was confusing. The problem wasn't with my implementation however but the choice of correspondence points which was an easy fix thankfully
